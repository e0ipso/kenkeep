# `ENTRY.md`

The entry catalog: the always-injected launchpad at `.ai/kenkeep/ENTRY.md`. Generated by `generateIndex` / `renderRootCatalog` (`src/lib/index-gen.ts`), a pure function of `nodes/`. No LLM involvement.

Unlike a per-folder `index.md` (which rolls up only its own direct leaves, and at the root would render a misleading `0 node(s)` header), `ENTRY.md` is purpose-built and carries only:

- whole-tree totals (node count, branch count, estimated tokens), and
- the top-level branch list with compact rollup counts (the next descent step).

It deliberately carries no leaf bodies and no topic index, so the always-injected payload stays bounded by branch count, not total node count. Cross-cutting topic recall is left to descent: each branch `index.md` rolls up its own tags under `## By topic`.

Carries frontmatter validated by `IndexFrontmatterSchema`:

```yaml
schema_version: 2
nodes_hash: sha256:<hex>
node_count: <whole-tree total>
```

`nodes_hash` is the GLOBAL content-addressed hash of the `nodes/` tree (see `map-nodes-hash`). Drift between the frontmatter hash and the live hash is how `doctor` and `kk-session-start` detect a stale entry catalog.

Regeneration paths:

- End of every `curate` run.
- The `npx kenkeep index rebuild` command (which also removes a legacy `INDEX.md` from before the rename).
- `index rebuild --stage` from a project's own pre-commit hook (e.g. via lint-staged), which also runs `git add` so the regenerated `ENTRY.md` lands in the same commit as the `nodes/` change.

`kk-session-start` injects the `ENTRY.md` body into every new AI session via the harness's `additionalContext` channel (or, on OpenCode, by writing to `.opencode/AGENTS.md`; on Copilot, into `.github/copilot-instructions.md`). Repos seeded before the rename are read from the legacy `INDEX.md` until the next rebuild.
