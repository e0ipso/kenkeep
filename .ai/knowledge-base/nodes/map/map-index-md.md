# `INDEX.md`

Deterministic catalog of every node. Generated by `generateIndex` (`src/lib/index-gen.ts`), a pure function of `nodes/` plus an injected `now`. No LLM involvement.

Carries frontmatter validated by `IndexFrontmatterSchema`:

```yaml
schema_version: 1
nodes_hash: sha256:<hex>
node_count: 47
```

`nodes_hash` is the content-addressed hash of the `nodes/` tree (see `map-nodes-hash`). Drift between the frontmatter hash and the live hash is how `doctor` and `kb-session-start.mjs` detect a stale INDEX.

Regeneration paths:

- End of every `curate` run.
- The `npx @e0ipso/ai-knowledge-base index rebuild` command.
- `index rebuild --stage` from a project's own pre-commit hook (e.g. via lint-staged), which also runs `git add` so the regenerated INDEX lands in the same commit as the `nodes/` change.

`kb-session-start.mjs` injects the INDEX body into every new AI session via the harness's `additionalContext` channel (or, on OpenCode, by writing to `.opencode/AGENTS.md`).
