# Harden the launch path and promote it into a canonical named launcher

## Objective
Guarantee that a long-running, non-context hook frees the host slot **before**
any host-dependent or unbounded operation, so no harness â€” current or future â€”
can kill it before it detaches. In the same pass, replace the drain-specific
detach mechanism with one canonical, generically named async-hook launcher
reusable by any long-running advisory hook. The harness-agnostic test from
Task 1 must pass after this change.

## Skills Required
- `typescript` â€” refactor `hook-detach.ts`, `hook-entry.ts`, and `stdin.ts` and
  rename the env/marker contract.
- `node-process` â€” reorder/guard the detached-spawn sequence (`detached`,
  `stdio: 'ignore'`, `unref()`), process-group isolation, and stdin/EOF handling.

## Acceptance Criteria
- [ ] The launch path detaches and returns before any host-dependent or
      unbounded operation; the Task 1 held-open-stdin/no-EOF test passes for all
      detach-reliant harnesses (Codex, Cursor, Copilot).
- [ ] The drain-specific `KENKEEP_DRAIN_DETACHED` env marker and any drain-only
      doc comments are removed; a neutral, launcher-oriented name replaces them.
      A tree-wide grep for `KENKEEP_DRAIN_DETACHED` returns zero occurrences.
- [ ] The capability is presented as a first-class named launcher (not an
      incidental `detach: true` flag); the payload-carrying env var and the
      child-identification marker are part of the launcher's defined contract.
- [ ] The launcher carries (or has the child acquire) whatever payload the hook
      needs; the drain's required payload (e.g. `cwd`) is preserved end to end.
- [ ] The `KENKEEP_BUILDER_INTERNAL` recursion guard and the detached child's
      process-group isolation are preserved and covered by tests.
- [ ] Claude (native async) and OpenCode (plugin async) mechanisms are not
      regressed; the full existing test suite passes.
- [ ] A launcher-contract test asserts both fast parent return AND that the
      detached child runs the work to completion (no fast-return-that-skips-work).

## Technical Requirements
- Files: `src/lib/hook-detach.ts`, `src/lib/hook-entry.ts`, `src/lib/stdin.ts`,
  and `src/harnesses/types.ts` (`HookSpec`) as needed to express async hooks.
- Current behavior to fix: `runHookEntry` does `raw = isDetachedChild() ?
  detachedPayload() : await readStdin(); if (!isDetachedChild() &&
  detachSelf(raw)) return;` â€” the unbounded `readStdin()` precedes `detachSelf`.
- Implementation options (choose per Task 1's confirmed cause): detach first and
  let the child acquire its payload independently; OR bound the pre-detach read
  with a short, timeout-safe deadline and an EOF-independent fallback while still
  carrying the available payload to the child. Reconcile the drain hook's absent
  `deadlineMs` with the launcher's own escape semantics.
- Keep the launcher a thin wrapper over the proven detached-spawn idiom; do not
  introduce a new dependency or heavier abstraction (YAGNI).

## Input Dependencies

## Output Artifacts

   diagnostic output), not merely that the parent exits fast.
