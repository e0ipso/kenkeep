# Baseline Scope and Helper Packaging

## Objective
Establish the editable source-of-truth files, behavior contract baseline, and safe location/reference strategy for the shared kk harness-detection helper before any conciseness rewrites.

## Skills Required
Use `repo-analysis` to identify generated versus source assets and packaging paths, and `markdown-audit` to extract the behavior-bearing contract from the scoped skills and prompt.

## Acceptance Criteria
- [ ] All files explicitly listed in plan 54 are inspected, plus canonical `src/templates-source/skills/kk-*`, `src/templates-source/prompts/proposal-extract.md`, generated `templates/` counterparts, and installed kk skill copies under every present harness directory (`.claude`, `.agents`, `.cursor`, `.opencode`, `.github` if present).
- [ ] A concise baseline checklist exists in the implementation notes, commit message draft, or working notes covering CLI primitives, flags, action types, `home_folder`, field names, output schemas, and conflict outcomes that must survive the rewrite.
- [ ] The implementation path for `kk-detect-harness.mjs` is decided and justified against the self-contained skill/packaging constraint, including first-install, `init --upgrade`, and `scripts/lint-detect-harness.mjs` impacts.
- [ ] Initial `wc -l` counts are captured for every scoped skill and prompt file, including canonical source and installed copies that will be edited or refreshed.
- [ ] Prompt and skill `Version:` comments that may need bumping are identified before edits begin.

Use your internal Todo tool to track these and keep on track.

## Technical Requirements
Inspect the scoped markdown files, the repository's generated-template conventions, schema source, detector drift lint, install/upgrade paths, and any relevant source-template copies before editing. Use `rg`, `wc -l`, and direct file reads. Do not edit generated `templates/` output directly.

## Input Dependencies
None.

- Baseline contract checklist for tasks 2, 3, and 4.
- Initial line-count snapshot.
- Packaging/source-of-truth decision for shared kk helper files.
- Versioning and changelog checklist for prompt and skill edits.
