# Rewrite all existing node files to match the slim schema

## Objective

Bring the 41 node files currently in `.ai/knowledge-base/nodes/practice/` and `.ai/knowledge-base/nodes/map/` in line with the new `NodeFrontmatterSchema`. Drop the five removed frontmatter fields from every file. Where a node body describes the supersession feature itself (notably `practice-determinism-contract.md`, which lists `valid_from`/`valid_until` as part of the determinism scope, and any node whose prose names `supersedes` / `superseded_by` / `valid_*` semantics), rewrite the prose to describe the current design. No retrospective framing; write as if the new design had always been the design.

## Skills Required

- kb-internals: edit knowledge-base node files with awareness of the frontmatter schema, body conventions, and project style rules.

## Acceptance Criteria

- [ ] Every file under `.ai/knowledge-base/nodes/practice/` and `.ai/knowledge-base/nodes/map/` parses cleanly against the slim `NodeFrontmatterSchema` (no missing, no extra fields).
- [ ] `grep -rnE "^(valid_from|valid_until|updated|supersedes|superseded_by):" .ai/knowledge-base/nodes` returns empty.
- [ ] `practice-determinism-contract.md` body no longer lists `valid_from` / `valid_until` among the inputs / contracts of the determinism contract.
- [ ] Any node body that referenced `supersedes`, `superseded_by`, "Recently superseded", or "valid window" semantics has been rewritten to describe the current design.
- [ ] No retrospective framing ("previously", "we used to", "earlier the schema included") appears in any rewritten body.
- [ ] No em-dash (`—`), en-dash (`–`), or ` - ` separator appears in any rewritten body. Acceptable inside fenced code blocks or YAML examples; not in narrative prose.
- [ ] Node ids, titles, kinds, tags, edges, confidence, summary fields are unchanged. This task only removes fields and rewrites supersession-related prose; it does not retitle nodes or change graph edges.
- [ ] If a node file existed only to document the supersession feature (no such node exists per current grep; verify), it is deleted; otherwise no node files are deleted.

Use your internal Todo tool to track these and keep on track.

## Technical Requirements

- Files: all `*.md` files under `.ai/knowledge-base/nodes/practice/` and `.ai/knowledge-base/nodes/map/`. Confirmed count at plan time: 18 practice + 23 map = 41 nodes.
- The frontmatter is YAML between the leading `---` and the next `---`. Remove exactly the five field lines wherever they appear.
- `INDEX.md` and `GRAPH.md` are regenerated by a later task; do not edit them in this task.

## Input Dependencies

- Task 1 completed (so the cleaned files will validate against the schema you are targeting).

## Output Artifacts

- 41 rewritten node files with trimmed frontmatter; a subset (those naming the removed fields in their body) additionally have prose rewritten.

## Implementation Notes

<details>
<summary>Step-by-step implementation guidance</summary>

1. **Inventory the body references first.** Before editing frontmatter wholesale, run:
   ```
   grep -rn "valid_from\|valid_until\|supersedes\|superseded_by\|Recently superseded\|valid window\|archival" .ai/knowledge-base/nodes
   ```
   The plan calls out at least `practice-determinism-contract.md` and `map-index-and-graph-files.md`. Note: `map-index-and-graph-files.md` is also referenced in the docs task (task 7); coordinate to avoid double-editing — for this task, only adjust if its body names the removed fields or supersession semantics. Otherwise leave to task 7. If both tasks need to edit it, this task edits the body; task 7 leaves it alone.

2. **Frontmatter trim.** For every node file:
   - Open the file, find the frontmatter block.
   - Delete the lines beginning with `valid_from:`, `valid_until:`, `updated:`, `supersedes:`, `superseded_by:`.
   - Preserve the order and indentation of the remaining lines.
   - Leave the body untouched in this pass.
   A simple scripted approach is acceptable: a `node` or `python` one-liner that walks the directory and strips the five keys from each YAML block. If you use a script, verify the diff before committing.

3. **Body rewrites.** For each file flagged in step 1:
   - **`practice-determinism-contract.md`**: locate the list of inputs / fields that the determinism contract covers. Remove `valid_from` and `valid_until` from that list. If the surrounding sentence depends on them, rewrite it to enumerate the surviving fields (`id`, `title`, `kind`, `tags`, `derived_from`, `relates_to`, `depends_on`, `confidence`, `summary`, body). Keep the contract's claim (`INDEX/GRAPH and nodes_hash are deterministic and content-addressed`) intact.
   - For any other node whose body mentions `supersedes`, `superseded_by`, `valid_from`, `valid_until`, "Recently superseded", or the supersession feature: rewrite the relevant sentence so it describes the current design (current nodes only on disk; replace-or-reject conflict model; git history is the audit trail). Do not narrate that something was removed.

4. **Style sweep on rewritten paragraphs.** Run on every file you touched in step 3:
   ```
   grep -nE "( - |—|–)" <file>
   ```
   Replace any hit in narrative prose with commas, colons, or parentheses. Hits inside fenced code blocks are acceptable.

5. **Validation.** After all edits:
   ```
   grep -rnE "^(valid_from|valid_until|updated|supersedes|superseded_by):" .ai/knowledge-base/nodes
   ```
   Must return empty. Then run (after task 3 completes for the type definitions, but you can run the read-side validation independently):
   ```
   npx ai-knowledge-base index rebuild --dry-run
   ```
   to confirm the slim schema accepts every file. If any file fails validation, fix the offending line.

6. **Do not** regenerate `INDEX.md` or `GRAPH.md` here. That belongs to the final verification task.

7. **Do not** edit anything under `.ai/knowledge-base/_sessions/`, `_logs/`, `_proposals/`, or `.state/`. Those are ephemeral capture and runtime state, not nodes.

</details>
